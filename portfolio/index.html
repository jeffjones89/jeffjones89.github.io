A chronological journey through my four projects at General Assembly's Web Development Immersive June - October, 2015. Prior to General Assembly I had 1-2 months of part time, self-taught experience, but had never built a full scale application prior to taking the course. Many more projects and code samples can be found on my GitHub profile.

Blackjack



My first project! It's funny to look back on this one and think how far I (and the rest of the WDI students) have come, relatively speaking, in the 9 weeks since it was due. SeedMe



SeedMe was the second project I built during WDI, during week 6 of the program. It's a Ruby on Rails application, designed to allow start-ups to post video links of their pitches, and connect and collaborate with other entreprenuers. SeedMe has hand-rolled user authentication, including a custom helper method for current_user, a liking and favoriting system using, with likes using the Acts As Votable gem and a rudimentary messaging system to allow users to send private messages to each other on the platform. I included random clips from movies that I enjoy as dummy data. Link to code can be found here. Technologies include:

  • Ruby |
  • Rails |
  • ActiveRecord

Menu Builder



Menu Builder is a group application (with a totally original and creative name!) built during Week 9 of General Assembly's Web Development Immersive. This is a single page application that utilizes the Nutritionix API to allow users to calculate nutritional value of a meal searching by restaurant or item. My role on the team was co-lead of the project, primarily focusing on the AJAX calls, handrolled user interface and object oriented JavaScript principles. Also point person on syncing up saving meals to the database. Link to code can be found here. Technologies include:

  • JavaScript/JQuery |
  • Express |
  • NodeJS |
  • Sequelize |
  • AJAX

Shots on Franklin



ALERT: Still tinkering! Shots on Franklin is an homage to my time in Chapel Hill/pet project that I decided to undertake as my final project in the Web Development Immersive. One of my favorite sites for advanced college basketball stats went behind a paywall so I figured I'd try and recreate what I could. It is a MEAN stack application that ties D3 data visualization to a custom AngularJS directive. Going into the week, I

  • had never built a restful API using MongoDB
  • had never built a fullstack MEAN application
  • had extremely limited exposure to D3js
I was able to get up and running with what I considered an MVP within the week. I'd like to keep playing around with the D3, and use this as a bit of a sandbox moving forward to tinker with Mongo and the intersection of a library (D3) and a framework (Angular) both built on the principle of binding data to the DOM.
